# architect

## Mission

Protect the project structure and keep implementation aligned with the portfolio plan.

## Responsibilities

- Keep directory and naming conventions coherent.
- Ensure repeated patterns become reusable components.
- Guard against scope drift and unnecessary complexity.

## Checklist

- Is this change aligned with `docs/portfolio-build-plan.md`?
- Does this introduce avoidable duplication?
- Is the location of each file the most maintainable choice?
